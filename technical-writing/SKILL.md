# Technical Writing

Transform complex information into engaging docs that people actually read.

## Workflow
1. **Audience Analysis**: Determine if the reader is a developer, PM, or end-user.
2. **Structural Clarity**: Use clear heading hierarchies (H1, H2, H3).
3. **Example-Driven**: Accompany every concept with a code snippet or flow chart.

## Writing Standards
- **Tone**: Professional and objective. Avoid "simply", "obviously", or "easy".
- **Formatting**: Code blocks must have language tags; define terms on first use.
- **Verification**: Check spelling, grammar, and link validity.
