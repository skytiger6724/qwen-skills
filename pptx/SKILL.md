# PPTX 技能規範 (專業美學與視覺設計)

## 核心目標
* 協助使用者建立具備高度專業感與美學的簡報架構。
* 確保簡報內容邏輯嚴密，且符合特定的視覺設計規範。
* 將繁雜的資料轉化為易於理解的視覺化語言。

## 視覺與排版規範 (嚴格遵守)

### 1. 配色與背景
* **核心配色**：嚴格遵守『紅、白』配色系統。
* **主色調**：紅色 (Hex: `FF3B30`)。
* **背景**：必須為純白色 (`FFFFFF`)。
* **運用原則**：60-70% 的視覺重量由紅色主導（如標題、重點形狀、圖表系列），搭配純白背景。避免使用雜亂的過渡色。

### 2. 字體規範
* **中文字體**：必須使用『微軟正黑體 (Microsoft JhengHei)』。
* **英數字體**：必須使用『Arial』。
* **字號建議**：
    * 投影片標題：36-44pt (粗體)
    * 內容標題/區塊頭：20-24pt (粗體)
    * 正文文字：14-16pt
    * 備註/引用：10-12pt

### 3. 插圖與視覺元素
* **插圖風格**：必須指定為『扁平化科技風格 (Flat Tech Illustration)』。
* **元素特徵**：具備圓角設計、磨砂玻璃感 (Frosted Glass) 或簡潔的線條感。
* **圖形要求**：每個投影片都必須包含至少一個視覺元素（圖表、圖示或形狀）。嚴禁純文字投影片。

### 4. 輸出格式
* **投影片比例**：固定為 `16:9` 橫向佈局。

## 行為與語言規則
* **語言**：全程使用**繁體中文**。
* **邏輯架構**：採用「三明治結構」——標題與結論頁使用強化視覺，內容頁保持簡潔專業。
* **排版禁忌**：
    * 嚴禁在標題下方使用強調線（這是 AI 生成的常見特徵）。
    * 嚴禁正文文字居中對齊（僅標題可居中）。
    * 嚴禁使用低對比度的元素（如淺灰字配白底）。

## 品質保證 (QA) 程序

**務必假設初次生成的結果存在問題。**

### 視覺 QA 檢核清單：
1. **配色**：是否只有紅與白？背景是否為純白？
2. **字體**：中文是否為微軟正黑體？英數是否為 Arial？
3. **重疊**：文字是否穿過形狀？元素是否堆疊錯誤？
4. **溢出**：文字是否超出投影片邊緣或文字框？
5. **對齊**：欄位或相似元素是否保持一致的對齊？
6. **對比度**：紅色元素在白底上是否清晰易讀？

### 驗證循環：
1. 生成簡報 -> 轉換為圖片 -> 視覺檢查。
2. 列出所有發現的問題（即便微小）。
3. 修復問題。
4. 重新驗證受影響的投影片，直到全篇無誤。

## 依賴項
* `pip install "markitdown[pptx]"` - 文字提取
* `npm install -g pptxgenjs` - 從頭建立簡報
* LibreOffice (`soffice`) - PDF 轉換
* Poppler (`pdftoppm`) - PDF 轉圖片
